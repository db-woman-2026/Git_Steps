# Step 13. 브랜치 만들고 합치기 반복 연습

## 목표

브랜치 생성, 수정, commit, main 복귀, merge 흐름을 혼자 반복합니다.

## 실습 순서

1. 새 브랜치를 만듭니다.
2. `practice` 폴더의 파일 하나를 수정합니다.
3. 변경 내용을 commit 합니다.
4. `main`으로 돌아옵니다.
5. 방금 만든 브랜치를 merge 합니다.
6. 필요하면 사용이 끝난 브랜치를 삭제합니다.

## VSCode에서 볼 것

- 현재 브랜치가 어디인지 계속 확인합니다.
- commit하지 않은 변경이 남아 있으면 브랜치 이동이나 merge 전에 정리합니다.

## 완료 기준

같은 흐름을 최소 두 번 반복하고, `main`에 모든 변경이 합쳐져 있으면 완료입니다.
