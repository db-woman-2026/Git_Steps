# Step 5. 새 파일을 추가한 브랜치 합치기

## 목표

브랜치에서 새 파일을 만들고 `main`에 합치는 상황을 연습합니다.

## 실습 순서

1. 새 브랜치 `branch/add-diary`를 만듭니다.
2. `practice/diary.md` 파일을 새로 만듭니다.
3. 짧은 일기 문장을 작성하고 commit 합니다.
4. `main`으로 돌아갑니다.
5. `branch/add-diary`를 merge 합니다.

## VSCode에서 볼 것

- 새 파일은 Source Control에서 추가된 파일로 표시됩니다.
- 다른 사람이 같은 이름의 파일을 만들지 않았다면 보통 충돌 없이 합쳐집니다.
- 이 단계도 fast-forward로 끝날 수 있습니다. merge 후 `main`에 새 파일이 보이면 정상입니다.

## 완료 기준

`main` 브랜치에 `practice/diary.md` 파일이 생기면 완료입니다.
