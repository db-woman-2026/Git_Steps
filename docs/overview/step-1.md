# Step 1. 첫 파일 수정 commit 만들기

## 목표

Git에서 파일을 수정한 뒤 그 변경을 commit으로 저장하는 흐름을 이해합니다.

## 실습 흐름

1. `practice/intro.md` 파일을 엽니다.
2. 자기소개 문장을 한 줄 추가합니다.
3. VSCode Source Control에서 변경된 파일을 확인합니다.
4. 변경 파일을 stage 합니다.
5. `Add my intro` 같은 짧은 메시지로 commit 합니다.

## VSCode에서 볼 것

- Source Control 아이콘에 숫자가 생기는지 확인합니다.
- 파일을 클릭하면 변경 전과 변경 후가 비교되는지 확인합니다.
- commit 후 변경 목록이 비워지는지 확인합니다.

## 완료 기준

`practice/intro.md` 수정 내용이 하나의 commit으로 저장되어 있으면 완료입니다.
