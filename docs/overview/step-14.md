# Step 14. 최종 결과 GitHub에 push하기

## 목표

step 0에서 연결한 GitHub 저장소에 지금까지 만든 최종 실습 결과를 다시 push하고 온라인에서도 확인합니다.

## 실습 흐름

1. `main` 브랜치에 있는지 확인합니다.
2. commit하지 않은 변경이나 conflict가 남아 있지 않은지 확인합니다.
3. `origin`이 내 개인 GitHub 저장소 주소인지 확인합니다.
4. `main` 브랜치를 push 합니다.
5. GitHub 웹사이트에서 최종 파일과 commit 기록을 확인합니다.

## VSCode에서 볼 것

- 원격 저장소는 온라인 백업 장소로 이해합니다.
- push 후 GitHub에서 같은 파일이 보이는지 확인합니다.
- 수강생 실습에서는 `main`만 push해도 충분합니다. `step-0`, `step-1` 같은 브랜치는 강사용 참고 브랜치입니다.

## 완료 기준

GitHub에서 `README.md`, `docs/overview`, `practice` 폴더와 commit 기록이 보이면 완료입니다.
